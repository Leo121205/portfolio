Contexte & Objectifs

Ce projet a été réalisé en binôme dans le cadre du module Conception et Développement d'Application du BTS SIO 1ère année. L'objectif était de créer un site e-commerce fonctionnel pour une agence de voyage fictive appelée VoyageHub.

Mohamed Mcirdi a pris en charge le design et la structure HTML. Léo Bigote a développé la base de données, la connexion PDO et les pages PHP dynamiques.

Objectifs pédagogiques

  • Concevoir une base de données relationnelle (MCD → MLD → SQL)
  • Connecter une application PHP à MySQL via PDO avec requêtes préparées
  • Générer des pages dynamiques depuis une base de données
  • Mettre en place un système de commande complet avec calcul métier
  • Travailler en binôme avec Git et résoudre des conflits de merge

Architecture

Frontend

  • HTML5 sémantique — structure accessible et bien organisée
  • CSS3 externe avec responsive design (media queries 768px / 480px), animations et transitions
  • JavaScript ES6 — filtre budget dynamique, calcul prix en temps réel, code promo côté client

Backend

  • PHP 8 avec connexion PDO sécurisée
  • Requêtes préparées anti-injections SQL
  • Validation double : côté client (JS) + côté serveur (PHP)

Base de données (MySQL)

  • 5 tables relationnelles : client, destination, voyage, commande, ligne_commande
  • Clés primaires et étrangères, relation n,n via table d'association ligne_commande
  • Transactions PDO (commit / rollback) pour les insertions en BDD

Étapes de réalisation

  1. Conception BDD — Modélisation MCD et MLD

    Création du schéma SQL avec 5 tables relationnelles et clés étrangères. Modélisation de la relation n,n entre commandes et voyages via la table d'association ligne_commande.

  2. Connexion PDO sécurisée

    Mise en place de la connexion à MySQL avec gestion des erreurs (try/catch), requêtes préparées anti-injections SQL et modes de fetch adaptés.

  3. Catalogue dynamique

    Remplacement des 3 cartes HTML statiques par un catalogue PHP chargeant les voyages depuis la BDD avec jointure SQL. Ajout d'un filtre budget côté JavaScript.

  4. Pages produit dynamiques

    Chaque voyage a sa propre page via produit.php?id=X avec calcul du prix en temps réel selon le nombre de personnes saisies.

  5. Système de commande complet

    Formulaire 8 champs avec validation double (JS + PHP), calcul métier : frais de dossier 29 €, code promo BTS2026 (-10%). Insertion en BDD dans 3 tables via transaction PDO (commit / rollback).

  6. CSS externe responsive

    Externalisation de tout le CSS inline dans un fichier style.css avec animations, transitions et media queries (768px / 480px).

  7. Déploiement Git en binôme

    Push sur GitHub, travail en branches parallèles et résolution de conflits de merge lors des intégrations.

Difficultés rencontrées

  • Conflit entre deux serveurs MySQL — XAMPP (port 3307) vs MySQL80 (port 3306). Résolu en désactivant MySQL80 dans les services Windows et en reconfigurant my.ini.
  • Modélisation du MCD — conception de la relation n,n entre commandes et voyages via la table d'association ligne_commande.
  • Conflits Git — travail en parallèle sur les mêmes fichiers lors de la phase d'intégration.
