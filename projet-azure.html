Infrastructure Cloud Azure

Déploiement d'une infrastructure hybride sur Microsoft Azure : VMs, réseaux virtuels, Azure AD Connect et sauvegarde cloud.

Contexte & Objectifs

Ce projet explore le déploiement d'une infrastructure hybride : une partie des ressources reste sur site (on-premise, Active Directory local efrei.local) et une autre est migrée vers Microsoft Azure. L'objectif est de comprendre les concepts fondamentaux du cloud (IaaS, PaaS) et la synchronisation des identités entre l'AD local et Azure AD (Entra ID).

Le projet a été réalisé avec un abonnement Azure for Students (100€ de crédits gratuits).

Architecture hybride

  • On-premise : DC01-EFREI (AD local), postes utilisateurs Windows 10
  • Azure : 2 VMs (Windows Server + Linux), VNet, Azure AD, Azure Backup
  • Synchronisation : Azure AD Connect entre AD local et Entra ID

Étapes de réalisation

  1. Configuration du tenant Azure et groupe de ressources

    Création du tenant Azure avec le compte EFREI (Azure for Students). Création d'un groupe de ressources "RG-EfreiCorp" dans la région France Central. Configuration des balises (tags) pour le suivi des coûts.

  2. Déploiement du réseau virtuel (VNet)

    VNet "VNET-EfreiCorp" : espace d'adressage 10.1.0.0/16. Sous-réseaux : Subnet-Servers (10.1.1.0/24), Subnet-Web (10.1.2.0/24). Groupes de sécurité réseau (NSG) : règles entrantes pour RDP (port 3389) et SSH (port 22) depuis mon IP uniquement.

  3. Déploiement des machines virtuelles Azure

    VM-WIN-SRV : Windows Server 2022, taille B2s (2 vCPU, 4 Go RAM), dans Subnet-Servers. VM-LINUX : Ubuntu 22.04 LTS, taille B1s (1 vCPU, 1 Go RAM), dans Subnet-Web. Disques gérés Standard SSD pour les deux VMs.

  4. Configuration Azure AD Connect

    Installation d'Azure AD Connect sur DC01-EFREI. Configuration Express (synchronisation complète de l'AD local vers Entra ID). Première synchronisation : 20 utilisateurs et 5 groupes importés dans Azure AD. Test de la SSO (Single Sign-On) avec un compte synchronisé.

  5. Configuration d'Azure Backup

    Création d'un coffre Recovery Services "RSV-EfreiCorp". Activation de la sauvegarde pour VM-WIN-SRV : planification quotidienne à 02h00, rétention 7 jours. Test de restauration : snapshot d'un disque de VM en 12 minutes.

  6. Surveillance avec Azure Monitor

    Configuration des alertes : CPU > 80%, RAM > 85%, espace disque < 10%. Tableau de bord personnalisé dans le portail Azure avec les métriques clés des VMs. Activation des logs diagnostics vers un compte de stockage.

Résultats

2VMs Azure déployées
20Utilisateurs synchronisés
1Backup configuré
3Alertes Monitor

L'infrastructure hybride est opérationnelle. Les utilisateurs du domaine local peuvent se connecter aux ressources Azure avec leurs identifiants habituels (via AAD Connect). Azure Monitor surveille en temps réel les métriques des VMs.

Ce que j'ai appris

Concepts cloud : IaaS vs PaaS vs SaaS, modèle de responsabilité partagée

Architecture réseau Azure : VNet, sous-réseaux, NSG, IP publiques/privées

Synchronisation identités AD local ↔ Entra ID avec Azure AD Connect

Gestion des coûts Azure et optimisation via les tailles de VMs et les alertes budget
