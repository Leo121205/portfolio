Contexte & Objectifs

Dans le cadre du BTS SIO option SISR, j'ai déployé une infrastructure Active Directory complète simulant l'environnement informatique d'une PME fictive nommée "EfreiCorp". L'objectif était de maîtriser la gestion centralisée des identités, des postes et des accès via un domaine Windows.

L'ensemble de l'infrastructure a été déployé en environnement virtuel sous VirtualBox, permettant de travailler en toute sécurité sans impact sur les systèmes de production.

Objectifs pédagogiques

  • Installer et promouvoir un contrôleur de domaine Windows Server 2022
  • Créer et structurer une arborescence d'Unités d'Organisation (OU)
  • Gérer les comptes utilisateurs, groupes de sécurité et permissions NTFS
  • Déployer des stratégies de groupe (GPO) pour la sécurisation des postes
  • Joindre des postes clients au domaine et valider l'authentification centralisée

Architecture

Plan d'adressage IP

  • Serveur contrôleur de domaine : 192.168.10.1/24 (IP statique)
  • Postes clients : 192.168.10.50–100 (DHCP géré par le serveur AD)
  • Nom de domaine : efrei.local
  • Réseau : Adaptateur Host-only VirtualBox (192.168.10.0/24)

Structure organisationnelle (OU)

  • OU_Direction — Directeurs et cadres (accès complet aux ressources partagées)
  • OU_Informatique — Administrateurs IT (membres du groupe Domain Admins local)
  • OU_RessourcesHumaines — Service RH (accès restreint aux dossiers confidentiels)
  • OU_Marketing — Équipe marketing (accès aux partages créatifs)
  • OU_Ordinateurs — Tous les postes joints au domaine

Étapes de réalisation

  1. Installation de Windows Server 2022 sur VirtualBox

    Création d'une VM avec 4 Go RAM, 2 vCPU, 60 Go disque. Installation de la version Evaluation de Windows Server 2022 (Desktop Experience). Configuration de l'adaptateur réseau Host-only.

  2. Configuration réseau et IP statique

    Attribution de l'IP 192.168.10.1/24, passerelle 192.168.10.1, DNS pointant sur lui-même (127.0.0.1). Renommage du serveur en "DC01-EFREI".

  3. Installation du rôle AD DS et DNS

    Via le Gestionnaire de serveur → Ajouter des rôles → Active Directory Domain Services + DNS Server. Installation des outils d'administration RSAT.

  4. Promotion en contrôleur de domaine

    Assistant de configuration : "Ajouter une nouvelle forêt" → Nom de domaine racine : efrei.local → Niveau fonctionnel Windows Server 2016 → Définition du mot de passe DSRM → Installation et redémarrage.

  5. Création de la structure OU

    Via Active Directory Users and Computers (ADUC) : création des 5 Unités d'Organisation. Application du principe de délégation administrative pour chaque OU.

  6. Création des comptes utilisateurs et groupes

    20 comptes utilisateurs répartis dans les OU. Groupes de sécurité : GRP_Admins, GRP_RH, GRP_Marketing, GRP_Direction. Attribution des appartenances et des permissions sur les partages réseau (NTFS + Partage).

  7. Configuration des GPO

    GPO_MotDePasse : longueur minimale 12 car., complexité activée, verrouillage après 5 tentatives. GPO_Bureau : fond d'écran imposé, désactivation du Panneau de configuration. GPO_Redirection : redirection du dossier Documents vers le partage réseau \\DC01-EFREI\Profils$.

  8. Jonction des postes clients au domaine

    Création d'une VM Windows 10 Pro. Configuration DNS pointant vers DC01-EFREI (192.168.10.1). Jonction au domaine efrei.local via Propriétés système → Modification du nom d'ordinateur → Domaine. Authentification réussie avec les comptes du domaine.

  9. Tests et validation

    Connexion avec différents comptes utilisateurs, vérification des GPO appliquées (gpresult /r), test de la redirection des dossiers, vérification des permissions NTFS, tests de verrouillage de compte.

Résultats

20 Comptes créés
5 OU structurées
3 GPO déployées
2 Postes joints

L'infrastructure Active Directory efrei.local est pleinement opérationnelle. Les utilisateurs s'authentifient sur le domaine depuis les postes clients, les GPO sont correctement appliquées (vérifiées via gpresult /r), et les redirections de dossiers fonctionnent comme attendu.

Ce que j'ai appris

Architecture et rôles d'un domaine Active Directory (DC, FSMO, catalogue global)

Conception d'une structure OU adaptée à l'organigramme d'une entreprise

Gestion fine des permissions : héritage NTFS, groupes imbriqués, principe du moindre privilège

Déploiement et débogage des GPO (gpupdate, gpresult, Event Viewer)

Intégration de postes clients et gestion des profils itinérants

Importance de la documentation pour la maintenabilité d'une infra AD
