Déploiement d'un serveur web sécurisé sur Linux Debian avec Apache, MariaDB, PHP, SSL/TLS et protection anti-intrusion Fail2ban.

Contexte & Objectifs

Ce projet consiste à déployer un serveur web LAMP (Linux, Apache, MariaDB, PHP) complet et sécurisé sur une VM Linux Debian. L'objectif est de maîtriser l'administration d'un serveur web en production : configuration des Virtual Hosts, sécurisation SSL/TLS avec Let's Encrypt, pare-feu UFW et protection contre le brute force avec Fail2ban.

Le serveur héberge une application web PHP simple (formulaire de contact avec base de données) pour démontrer l'intégration de la stack complète.

Étapes de réalisation

  1. Préparation du serveur Linux Debian 12

    VM VirtualBox, 2 Go RAM, 20 Go disque. Mise à jour complète, création d'un utilisateur admin non-root avec sudo. Configuration du SSH (désactivation login root, port 22 → authentification par clé recommandée).

  2. Installation et configuration d'Apache2

    Installation Apache2 + activation des modules : a2enmod rewrite headers ssl. Création d'un Virtual Host pour le site (ServerName efreicorp.local). Test de la configuration : apache2ctl configtest.

  3. Installation et sécurisation de MariaDB

    Installation, mysql_secure_installation, création de la base "efreicorp_db" et de l'utilisateur "webuser" avec droits limités (SELECT, INSERT, UPDATE sur la seule base applicative). Aucun accès distant autorisé.

  4. Installation de PHP 8.2 et déploiement de l'application

    Installation PHP 8.2 + extensions : php-mysql, php-curl, php-mbstring, php-xml. Déploiement de l'application PHP dans /var/www/efreicorp/ avec permissions 750 (www-data:www-data). Configuration du fichier .env pour les credentials DB (hors du DocumentRoot).

  5. Configuration du pare-feu UFW

    Politique par défaut : deny incoming, allow outgoing. Règles ouvertes : SSH (22/tcp), HTTP (80/tcp), HTTPS (443/tcp). Activation : ufw enable. Vérification : ufw status verbose.

  6. Certificat SSL avec Let's Encrypt / Certbot

    Installation de Certbot pour Apache. Génération du certificat pour le domaine (environnement lab avec domaine local + entrée hosts). Configuration HTTPS automatique, redirection HTTP → HTTPS. Renouvellement automatique via cron.

  7. Protection avec Fail2ban

    Installation Fail2ban, activation de la jail SSH (5 tentatives → ban 1h) et de la jail Apache (protection contre les scan de paths malveillants). Configuration du fichier jail.local. Test : simulation de brute force SSH → IP bannie automatiquement.

Résultats

Le serveur web est opérationnel, accessible en HTTPS avec certificat valide. L'application PHP communique correctement avec MariaDB. Fail2ban a été validé en banissant une IP après 5 tentatives SSH échouées simulées.

Maîtrise complète de la stack LAMP en production sur Linux Debian

Configuration avancée d'Apache2 : Virtual Hosts, modules, .htaccess, headers sécurité

Infrastructure SSL/TLS avec Let's Encrypt et renouvellement automatique

Durcissement Linux : UFW, Fail2ban, permissions fichiers, SSH hardening
