Contexte & Objectifs

Ce projet porte sur le déploiement d'une infrastructure WiFi professionnelle pour une entreprise fictive. L'enjeu est de proposer une connectivité sans fil sécurisée pour les employés (WPA2-Enterprise avec authentification RADIUS) et un accès limité pour les visiteurs (portail captif isolé).

La segmentation WiFi par VLAN garantit l'isolement entre les réseaux employés et invités, et la politique de sécurité est renforcée par l'authentification centralisée via FreeRADIUS couplé à l'Active Directory.

Architecture WiFi

  • SSID EfreiCorp-Staff : WPA2-Enterprise / EAP-TTLS — VLAN 20 (Employés)
  • SSID EfreiCorp-Guests : Portail captif, bande passante limitée — VLAN 100 (Invités)
  • Standard : 802.11ax (Wi-Fi 6) sur bandes 2.4 GHz et 5 GHz

Étapes de réalisation

  1. Analyse des besoins et plan de déploiement

    Relevé de couverture (heat map simulée) pour positionner les points d'accès. Estimation : 2 AP pour couvrir 3 salles (environ 300 m²). Choix du matériel : TP-Link EAP670 (Wi-Fi 6, dual-band). Documentation du plan de fréquences (canaux 1/6/11 pour 2.4 GHz, 36/52/100 pour 5 GHz).

  2. Installation et configuration de FreeRADIUS

    Installation sur le serveur Linux Debian (apt install freeradius freeradius-ldap). Configuration du module LDAP pour pointer vers DC01-EFREI (port 389). Définition des clients RADIUS (NAS) : adresses IP des points d'accès + secret partagé. Test avec radtest depuis le serveur.

  3. Configuration des points d'accès

    Via l'interface Omada Cloud (TP-Link) : création des deux SSIDs. SSID Staff : sécurité WPA2-Enterprise, serveur RADIUS 192.168.10.5:1812, attribution VLAN 20 dynamique. SSID Guests : portail captif avec page de bienvenue personnalisée, limitation débit 5 Mbps down / 2 Mbps up, VLAN 100.

  4. Configuration des VLANs côté switch

    Ports vers les AP configurés en trunk (VLANs 20 et 100 autorisés). VLAN 100 (Invités) routé vers Internet uniquement, accès au LAN interne bloqué par ACL. DHCP dédié pour le VLAN 100 (pool 172.16.0.0/24, durée de bail 4h).

  5. Tests de connectivité et sécurité

    Connexion employé avec identifiants AD → succès, attribution VLAN 20. Tentative de connexion avec identifiants invalides → refus. Connexion invité via portail captif → succès, accès Internet uniquement (ping vers 192.168.10.1 bloqué). Scan WiFi avec outil de détection de rogues APs.

Résultats

L'infrastructure WiFi est opérationnelle. L'authentification WPA2-Enterprise fonctionne correctement avec les identifiants Active Directory. Les réseaux Staff et Guests sont bien isolés, validé par des tests de connectivité croisés.

Ce que j'ai appris

Standards Wi-Fi 802.11 : évolution de 802.11ac à 802.11ax (Wi-Fi 6), OFDMA, MU-MIMO

Différences entre WPA2-Personal (PSK) et WPA2-Enterprise (802.1X/RADIUS)

Installation et configuration de FreeRADIUS avec backend LDAP/AD

Segmentation WiFi par VLAN et association SSID ↔ VLAN sur les AP

Plan de fréquences pour éviter les interférences entre AP (canaux non chevauchants)

Configuration d'un portail captif pour gérer l'accès invités de façon sécurisée
