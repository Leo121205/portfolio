Contexte & Objectifs

Ce projet consiste à simuler une infrastructure réseau d'entreprise complète en environnement virtualisé. L'objectif est de maîtriser la mise en œuvre du plan d'adressage IP, la segmentation réseau et l'accès à Internet via NAT, le tout sans matériel physique.

L'infrastructure reproduit les zones typiques d'une PME : LAN interne (postes utilisateurs), serveurs internes, et accès WAN/Internet via NAT VirtualBox.

Objectifs

  • Concevoir un plan d'adressage IP cohérent et documenté
  • Déployer et interconnecter plusieurs VMs sur différents segments réseau
  • Configurer l'accès Internet via NAT pour les VMs
  • Valider la connectivité et documenter les résultats (ping, tracert)

Plan d'adressage

Segments réseau

  • LAN Interne (Host-only) : 192.168.10.0/24 — Postes utilisateurs + Serveurs
  • WAN / Internet (NAT VirtualBox) : 10.0.2.0/24 — Accès Internet

Attribution des adresses

  • DC01-EFREI (Windows Server 2022) — 192.168.10.1/24 (statique, rôles AD+DHCP+DNS)
  • SRV-WEB (Linux Debian) — 192.168.10.10/24 (statique, serveur web Apache)
  • PC-USER01 (Windows 10) — 192.168.10.50 (DHCP depuis DC01)
  • PC-USER02 (Windows 10) — 192.168.10.51 (DHCP depuis DC01)

Étapes de réalisation

  1. Conception du plan réseau

    Schéma de l'infrastructure sur papier : identification des VMs, des rôles, des segments et du plan d'adressage IP. Documentation des flux attendus entre les différentes zones.

  2. Configuration des réseaux VirtualBox

    Création d'un réseau Host-only (192.168.10.0/24) dans VirtualBox Network Manager. Désactivation du DHCP intégré VirtualBox (géré par Windows Server). Chaque VM dispose de deux adaptateurs : Host-only (LAN) + NAT (Internet).

  3. Configuration des serveurs (IPs statiques)

    DC01 : IP 192.168.10.1, masque 255.255.255.0, DNS local (127.0.0.1). SRV-WEB Debian : édition de /etc/network/interfaces pour IP statique 192.168.10.10.

  4. Configuration du service DHCP

    Rôle DHCP installé sur DC01. Étendue : 192.168.10.50–100, masque /24, passerelle .1, DNS .1. Exclusion des IPs statiques (plage 192.168.10.1–49).

  5. Tests de connectivité interne

    Ping entre tous les hôtes du LAN. Tracert pour vérifier les chemins. Vérification de la résolution DNS (nslookup efrei.local). Test d'accès au partage réseau depuis les postes clients.

  6. Validation de l'accès Internet

    Test ping vers 8.8.8.8 et résolution de noms publics (ping google.com). Vérification du NAT VirtualBox actif sur l'interface WAN. Accès web depuis les postes clients via le navigateur.

Résultats

4 VMs déployées
2 Segments réseau
100% Connectivité validée

Toutes les VMs communiquent correctement sur le LAN interne. Les postes clients obtiennent bien leur adresse IP via DHCP depuis DC01. L'accès Internet fonctionne depuis tous les hôtes via le NAT VirtualBox.
