Toutes les manipulations décrites ici ont été réalisées dans un environnement lab entièrement isolé (réseau interne VirtualBox, aucune connexion au réseau de production). Les techniques présentées sont utilisées à des fins pédagogiques dans le cadre du BTS SIO SISR.

Contexte & Objectifs

Ce projet de cybersécurité a pour but d'identifier et exploiter des vulnérabilités courantes (réseau et web) afin de mieux comprendre les attaques et de proposer des contre-mesures efficaces. L'environnement est un réseau lab fermé composé de VMs Kali Linux (attaquant), Metasploitable 2 et DVWA (cibles).

Vulnérabilités étudiées

  • MITM (Man-in-the-Middle) : ARP Poisoning + capture de trafic
  • SQL Injection : Extraction de données depuis DVWA (niveaux Low à High)
  • XSS (Cross-Site Scripting) : Réfléchi et persistant
  • CSRF (Cross-Site Request Forgery) : Forçage d'actions non autorisées

Attaque 1 — MITM (ARP Poisoning)

L'attaque Man-in-the-Middle par ARP Poisoning consiste à envoyer de fausses réponses ARP pour associer l'adresse MAC de l'attaquant aux IPs de la victime et de la passerelle. Tout le trafic passe alors par l'attaquant.

Exploitation

  1. Activation du forwarding IP

    echo 1 > /proc/sys/net/ipv4/ip_forward — permet à Kali de relayer le trafic et d'éviter de couper la connexion de la victime.

  2. ARP Poisoning avec Ettercap

    Ettercap en mode graphique : sélection des cibles (victime 192.168.10.50, passerelle 192.168.10.1). Lancement de l'empoisonnement ARP : envoi de faux ARP Reply aux deux cibles.

  3. Capture de trafic avec Wireshark

    Capture du trafic HTTP en clair sur l'interface eth0. Filtres Wireshark : http.request.method == POST pour identifier les formulaires d'authentification. Extraction des credentials en clair depuis les paquets HTTP.

Contre-mesures recommandées

  • HTTPS/HSTS : Chiffrement TLS empêche la lecture du contenu intercepté
  • ARP inspection dynamique : Activation sur les switches Cisco (DAI)
  • Segmentation VLAN : Isolation des postes pour limiter la portée de l'attaque
  • VPN interne : Chiffrement de bout en bout même sur le LAN

Attaque 2 — SQL Injection

L'injection SQL exploite une absence de validation des entrées dans les requêtes SQL. Sur DVWA, le champ de recherche d'utilisateur est vulnérable à l'injection.

Exploitation (niveau Low)

  • Payload basique : 1' OR '1'='1 → retourne tous les utilisateurs
  • UNION-based : 1' UNION SELECT user,password FROM users-- - → extraction des hashes MD5
  • Cassage des hashes MD5 avec des tables arc-en-ciel (CrackStation)

Contre-mesures

  • Requêtes préparées (Prepared Statements) : Séparation stricte code/données
  • ORM sécurisé : Utilisation de frameworks avec protection intégrée
  • Moindre privilège : Compte DB avec droits SELECT uniquement
  • WAF : Web Application Firewall pour bloquer les payloads connus

Attaque 3 — XSS & CSRF

XSS Réfléchi

Injection d'un script dans un paramètre URL non filtré. Payload : <script>alert(document.cookie)</script> — exécution dans le navigateur de la victime, vol potentiel de session.

XSS Persistant

Injection d'un script dans un champ de commentaire stocké en base. Le script s'exécute pour chaque visiteur de la page. Démonstration : keylogger injecté capturant toutes les frappes clavier.

CSRF

Création d'une page HTML piégée qui soumet automatiquement un formulaire de changement de mot de passe en se faisant passer pour l'utilisateur connecté. Réussi sur DVWA niveau Low (sans token CSRF).

Contre-mesures

  • Content Security Policy (CSP) : Restreint les sources de scripts autorisées
  • Encodage des sorties : htmlspecialchars() en PHP pour neutraliser les balises
  • Tokens CSRF : Jeton unique par formulaire, vérifié côté serveur
  • SameSite cookies : Attribut SameSite=Strict sur les cookies de session

Ce que j'ai appris

Fonctionnement des attaques MITM et des protocoles réseau sous-jacents (ARP, TCP)

Techniques d'injection SQL et d'extraction de données depuis une base vulnérable

Importance du développement sécurisé dès la conception (Secure by Design)

Utilisation de Burp Suite pour intercepter et modifier des requêtes HTTP
